# 不拼单词

不拼单词是我为自己备考雅思做的一个单词拼写练习软件。

我一直很喜欢 “不背单词” 这类产品，它会根据你对单词的熟悉程度自动安排复习时间，几乎不用自己操心什么时候该复习。  
但在雅思备考里，我更在意的是“拼写”而不只是“认词”，现成软件里又很少有专门针对拼写训练的流程。  
所以我仿照这种产品思路，做了一个基于记忆曲线推送拼写练习的应用，并且支持根据自己的词书和背景继续扩展。

## 适合谁

- 正在备考雅思、托福或其他需要拼写能力的考试
- 想把“记住单词”变成“真的会拼”
- 喜欢简洁、轻量、自动安排复习节奏的个人工具

## 功能

- 支持 CSV 导入，并在导入时映射“单词”“释义”“音标”列
- 支持学习模式和复习模式
- 学习模式每次抽取 20 个未学习单词
- 复习模式抽取 `next_review` 到期的单词
- 支持 Enter 触发拼写判定
- 支持熟悉、模糊、不熟悉三种结果判定
- 支持拼写页退出并保存进度，下次继续同一组 20 词
- 支持本组总结和错词本显示

## 设计思路

- 参考的是“根据熟悉程度自动安排复习”的使用体验，但重点改成了拼写训练
- 整个界面尽量保持极简，减少干扰
- 代码主要通过 Qt 直接写 UI，方便我后续继续按自己的备考习惯调整

## 依赖

- Qt 6
- CMake 3.16+
- SQLite（使用 QtSql 模块）

## 编译

在项目根目录执行：

```bash
cmake -S . -B build
cmake --build build -j4
```

运行：

```bash
./build/VibeSpeller
```

如果 CMake 找不到 Qt 6，可以显式指定 Qt 路径：

```bash
cmake -S . -B build -DCMAKE_PREFIX_PATH="/path/to/Qt/6.x.x/macos"
```

## CSV 格式

导入时会先读取 CSV 第一行作为表头，然后让你手动映射列。

建议 CSV 至少包含以下列：

- 单词列
- 释义列
- 可选的音标列

示例：

```csv
word,translation,phonetic
tourism,旅游
nation,国家
```

## 数据库存储

程序会在本地创建 SQLite 数据库，并保存：

- 单词数据
- 复习间隔
- 下次复习时间
- 当前拼写会话进度

## 项目结构

## 说明

- 默认界面按照竖屏比例设计
- 所有 UI 主要通过代码创建，方便后续直接调整布局和样式
- 如果导入重复单词，程序会自动跳过
